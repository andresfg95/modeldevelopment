Section 02

Core Components

Six essential building blocks that every neural network relies on — each with a distinct role in turning raw data into accurate predictions.

🧱

01 / Layers

Layers

Layers are organized groups of neurons. The input layer receives raw features (e.g., X, Y coordinates). Hidden layers learn increasingly abstract representations. The output layer produces the final prediction or classification score.

Input → [Hidden₁, Hidden₂, …] → Output
⚡

02 / Neurons

Neurons

A neuron computes a weighted sum of its inputs, adds a bias, then passes the result through an activation function. Each neuron detects one feature or pattern. More neurons per layer = more capacity to represent complex patterns.

output = f( Σ(wᵢ · xᵢ) + b )
⚖️

03 / Weights

Weights

Weights are learnable parameters that scale each connection between neurons. A high weight amplifies a signal; a weight near zero suppresses it; a negative weight inverts it. Training adjusts weights to minimise prediction error.

w ← w − η · ∂L/∂w
🔀

04 / Activation Functions

Activation Functions

Without activation functions, stacking layers is equivalent to one linear layer. Activations introduce non-linearity, letting the network learn curves, spirals, and complex boundaries. Common choices: ReLU, Tanh, Sigmoid.

ReLU: f(x) = max(0, x) | Tanh: f(x) = (eˣ−e⁻ˣ)/(eˣ+e⁻ˣ)
📉

05 / Loss Functions

Loss Functions

The loss function measures how wrong the network's predictions are. For classification, cross-entropy loss is typical. For regression, mean squared error (MSE) is common. Lower loss = better predictions. The optimizer tries to minimise this value.

MSE = (1/n) Σ(ŷᵢ − yᵢ)² | CE = −Σ yᵢ log(ŷᵢ)
🧭

06 / Optimization Algorithms

Optimization Algorithms

Gradient Descent (and variants like SGD, Adam) iteratively adjusts weights in the direction that reduces loss. The learning rate controls step size — too large and it overshoots; too small and training is slow.

Adam: mₜ = β₁mₜ₋₁ + (1−β₁)gₜ / vₜ = β₂vₜ₋₁ + (1−β₂)gₜ²

Summary

Key Insights from the Playground

Hands-on experimentation with the TensorFlow Neural Network Playground revealed these critical lessons about how neural networks behave in practice.

INSIGHT 01

More Layers ≠ Always Better

Adding hidden layers helps with complex data (spiral, XOR) but can overfit simple datasets. Architecture must match problem complexity.

INSIGHT 02

Noise Hurts, But Is Real

High noise forces the model to generalise instead of memorise. Regularisation (L1/L2) and smaller networks handle noise better.

INSIGHT 03

Features Are Leverage

Adding X² or sin(X) as features can let a simple network solve problems that a deeper one struggles with. Feature engineering matters.

INSIGHT 04

Learning Rate Is Critical

A high learning rate causes oscillation and divergence; too low means slow convergence. 0.03 proved stable across most experiments.

INSIGHT 05

Activation Shapes Boundaries

Tanh and ReLU enable non-linear decision boundaries. Linear activations collapse the network to a single linear classifier regardless of depth.

INSIGHT 06

Loss Tells the Full Story

Watching training vs. test loss reveals overfitting early. Diverging test loss is the signal to reduce capacity or add regularisation.

Visualising neural networks makes abstract mathematics tangible. Seeing weights animate and decision boundaries shift in real time builds intuition that equations alone cannot provide.

06 — Key Takeaways
📈
Scale is Everything

Performance follows scaling laws (Chinchilla, 2022): more compute + more data = better models. But costs scale super-linearly — efficiency research (MoE, sparse models) is now critical.

🧭
Alignment is Non-Optional

Raw pre-trained models are capable but unsafe to deploy. RLHF and Constitutional AI are not afterthoughts — they determine whether a model is trustworthy enough for commercial use.

🌍
Sustainability Matters

A frontier training run can emit as much CO₂ as 500+ transatlantic flights. Smaller, data-efficient models (Phi-3, Mistral) are a direct industry response to this challenge.
